Process of Adding Code to an Existing Repository

Once a repository is created, we can continue to improve it by adding new code or updating existing files. The process generally involves three main steps:

  • Stage the updated files so Git knows which changes to track.
  • Commit the changes with a clear message that describes what was updated.
  • Push the changes to the remote repository (e.g., GitHub) so others can see and use the updated code.

Git Commands for Making Changes

1. Stage Changes

git add filename.html

Stage a specific file. To stage all changes at once:

git add .

2. Commit Changes

git commit -m "Update project page with new instructions"

The -m flag lets you add a message to explain what was changed.

3. Push Changes

git push origin main

This sends your committed changes to the remote repository on GitHub.
