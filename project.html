Why Remote Respositories Are Necessary

Remote repositories are important because they allow projects to be stored online and accessed from anywhere. They act as a backup and make collaboration possible, since multiple developers can work on the same project at the same time. They also keep a full version history to track changes easily.

Steps for Creating a Project on GitHub

  1. Log in to your GitHub account.
  2. Click on the “+” icon in the top right corner and select New repository.
  3. Enter a repository name (e.g., MyResponsiveProject).
  4. Choose visibility ( Public or Private.)
  5. Click Create repository.
  6. Copy the repository URL e.g., https://github.com/username/MyResponsiveProject.git

Git Commands Needed

1. Create a Project Locally


            mkdir MyResponsiveProject
            cd MyResponsiveProject
            git init 
            echo "<h1>My Responsive Project</h1>" > index.html
            git add index.html 
            git commit -m "Initial commit with index page"

2. Link a Local Repo to a Remote Repo

            git remote add origin https://github.com/username/MyResponsiveProject.git
            git push -u origin main

How Other People Can Contribute

  • Fork the repository on GitHub to create their own copy.
  • Clone the forked repository to their local computer.
  • Create a new branch and make changes.
  • Commit and push the changes to their forked repository.
  • Open a Pull Request (PR) to suggest merging their changes into the original project.
